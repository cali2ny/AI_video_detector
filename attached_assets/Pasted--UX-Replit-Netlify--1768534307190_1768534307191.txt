당신은 시니어 풀스택 웹 개발자이자 머신러닝·컴퓨터비전·UX 엔지니어입니다.
Replit 환경에서 개발하고, 최종 배포는 Netlify로 할 웹 서비스를 구현해야 합니다.

내가 만들고 싶은 서비스는 다음과 같습니다.

YouTube 영상 URL을 입력하면, 텍스트 메타데이터(제목/설명 등)가 아니라 **실제 화면 정보(썸네일/프레임)**와 딥러닝 기반 탐지 API를 이용해 “AI가 생성한 영상일 가능성”을 분석하고, 결과를 상업용 SaaS 제품처럼 고급스러운 UI로 보여주는 서비스입니다.
서비스 작업용 이름은 “AI Video Inspector”입니다.

중요한 제약:
이번 버전에서는 YouTube 링크만 입력받고, 파일 업로드는 제외합니다.
또한 영상 제목(title)은 탐지 알고리즘에서 완전히 제외해 주세요. 분석에는 화면 정보(이미지)와 필요하다면 길이·채널명 정도의 보조 메타데이터, 그리고 외부 딥러닝 API만 사용합니다.

전체 스택 요구사항은 다음과 같습니다.

프론트엔드: React + TypeScript + Vite
스타일: Tailwind CSS
백엔드: Netlify Functions (Node.js + TypeScript)
빌드: Vite 빌드 결과물이 dist 디렉터리에 생성되어 Netlify에서 그대로 호스팅 가능해야 합니다.
구성: 루트에 netlify.toml을 두고, Functions 디렉터리는 netlify/functions를 사용합니다.

[탐지 알고리즘 / 백엔드 설계]

백엔드의 핵심은 “탐지 알고리즘”입니다. 이 부분을 세 레이어로 설계해 주세요.

프레임 기반 휴리스틱/주파수 분석 레이어

외부 딥러닝 기반 AI/딥페이크 탐지 API 레이어

두 점수를 결합하고 사용자용 결과(점수/라벨/설명)를 만드는 결합 레이어

타입 정의는 src/types/analysis.ts에 따로 분리해 주세요. 타입은 다음과 같습니다.

AnalyzeVideoRequest:
videoUrl: string

Label: "LIKELY_AI" | "UNCLEAR" | "LIKELY_HUMAN"

AnalyzeVideoResponse:
score: number // 0~100 최종 점수 (AI 생성일 가능성)
label: Label
reasons: string[] // 사람이 읽을 수 있는 근거 설명
tips: string[] // 사용자가 직접 확인해 볼 팁
meta: {
sourceType: "YOUTUBE_URL";
videoId: string;
durationSeconds?: number;
channelTitle?: string;
thumbnailUrl?: string;
}
debug: {
heuristicScore: number; // 휴리스틱 기반 점수 (0~100)
externalApiScore: number | null; // 외부 딥러닝 API 점수 (0~100, 없으면 null)
finalScore: number;
}

분석 요청은 /api/analyze (Netlify Redirect를 통해 /.netlify/functions/analyze-video로 연결)로 POST JSON { videoUrl }를 받도록 해 주세요.

YouTube URL 처리 방식:

utils/extractVideoId.ts에 YouTube URL에서 videoId를 추출하는 순수 함수를 작성해 주세요.
일반적인 ?v= 파라미터, youtu.be/<id>, shorts URL 등을 가능한 한 커버해 주세요.

썸네일 URL은 https://img.youtube.com/vi/
<videoId>/maxresdefault.jpg 혹은 사용 가능한 해상도로 구성합니다. 이 URL을 화면 분석과 외부 API에 활용합니다.

YouTube 메타데이터는 선택 사항입니다.

환경 변수 YT_API_KEY가 있으면, YouTube Data API v3의 videos 엔드포인트를 호출해 duration, channelTitle 등을 가져옵니다.

없으면 videoId와 썸네일 URL만 사용하고 meta의 나머지 필드는 undefined로 두어도 됩니다.

프레임 기반 휴리스틱/주파수 분석 레이어

netlify/functions/analyze-video.ts 내부에 간단한 이미지 휴리스틱 분석 함수를 만들어 주세요. Node.js 환경에서 실행 가능한 수준에서, 다음과 같은 개념적인 특징을 계산하는 가벼운 로직이면 됩니다.

전체 밝기 분포 (너무 평균적/균일한지)

색상 히스토그램이 한두 색에 과도하게 치우쳐 있는지

엣지(경계선) 밀도: 에지를 단순하게 계산해서 텍스처가 거의 없는지, 혹은 과도하게 샤픈된 것 같은지

블록 노이즈/압축 아티팩트의 강도 등, 현실적인 범위에서 구현 가능한 수준의 간단한 통계

정교한 신호 처리가 아니라, “현실 촬영 영상에 비해 지나치게 매끈하거나, 너무 균일하거나, 인위적으로 샤프한 경우 점수를 올리는” 방향의 휴리스틱 점수만 있으면 됩니다.
이 함수는 썸네일 이미지(필요하다면 한 장만 사용)에서 이런 특징들을 계산하고, 0~100 사이의 heuristicScore를 반환합니다. 구현은 당신이 창의적으로 설계하되, 코드와 주석에서 어떤 특징을 보고 점수를 올리거나 내리는지 드러나게 작성해 주세요.

외부 딥러닝 기반 탐지 API 레이어

이 레이어는 실제로 딥러닝 모델을 직접 올리는 대신, 이미지를 받아 deepfake/AI-generated 여부를 확률로 반환해 주는 외부 REST API를 호출하는 구조입니다. 어떤 상용 서비스를 쓸지는 나중에 내가 결정할 것이므로, 지금은 “이미지 URL 리스트를 받고, { probability: number }를 반환하는 API”를 가정한 래퍼를 만들어 주세요.

환경 변수:
AI_DETECT_API_BASE_URL
AI_DETECT_API_KEY

함수 시그니처:
async function callExternalDetectionApi(frames: string[]): Promise<number | null>

구현 규칙:

BASE_URL 또는 API_KEY가 없으면 즉시 null을 반환합니다.

있으면, BASE_URL의 /detect-video 같은 엔드포인트에 POST 요청을 보냅니다.
Body 예시: { frames: string[] } (썸네일 URL 하나만 넣어도 되고, 확장 가능하게 설계)
Header: Authorization: Bearer <API_KEY>, Content-Type: application/json

응답은 { probability: number } 형태라고 가정하고, 0~1 사이의 probability를 0~100으로 변환해 externalApiScore로 사용합니다.

네트워크 오류나 응답 파싱 오류가 있으면 에러를 throw하지 말고 콘솔에 로깅 후 null을 반환합니다.

점수 결합 및 결과 생성 레이어

메인 핸들러에서 영상 URL을 받아서 다음 순서로 처리합니다.

videoUrl에서 videoId 추출

썸네일 URL 구성

(선택) YouTube API로 durationSeconds, channelTitle 가져오기

휴리스틱 분석 함수로 heuristicScore(0~100) 계산

callExternalDetectionApi로 externalApiScore(0~100 또는 null) 계산

두 점수를 결합해 finalScore 계산:

externalApiScore가 null이 아니면
finalScore = heuristicScore * 0.3 + externalApiScore * 0.7

externalApiScore가 null이면
finalScore = heuristicScore

finalScore는 0~100 범위로 clamp 해 주세요.

label은 finalScore 기준:
70 이상: "LIKELY_AI"
40 이하: "LIKELY_HUMAN"
중간: "UNCLEAR"

reasons 배열에는 최소 3~6개의 자연어 설명을 담습니다. 예를 들어, 휴리스틱 결과에 따라
“색상 분포가 매우 균등해 실제 촬영 영상보다 인위적인 느낌이 있습니다.”,
“고주파 성분이 상대적으로 낮아, 질감이 지나치게 매끈하게 보이는 경향이 있습니다.”
와 같이 “무엇을 봤고, 왜 AI 쪽으로/인간 쪽으로 판단했는지”를 설명해 주세요.
외부 API 결과가 있다면 “딥러닝 기반 탐지 모델이 이 영상을 약 X% 확률로 AI 생성으로 분류했습니다.” 같은 문장을 reasons 중 하나로 포함합니다.

tips 배열에는 사용자가 직접 영상을 확인할 때 도움이 되는 조언 3~4개를 넣어 주세요. 예:
“입 모양과 음성이 실제로 잘 맞는지 몇 군데 멈춰서 확인해 보세요.”
“프레임 사이에서 손가락, 악세서리, 글자가 깨지거나 변형되지 않는지 살펴보세요.”
“중요한 정보라면 다른 출처나 공식 채널과도 교차 확인해 보세요.”

debug에는 heuristicScore, externalApiScore, finalScore를 그대로 넣어 주세요.

Netlify Functions 쪽 파일 구조:
netlify/functions/analyze-video.ts (handler + 위 로직 포함)

netlify.toml에는 다음 내용을 포함해 주세요.

[build]
command = "npm run build"
publish = "dist"

[functions]
directory = "netlify/functions"

[[redirects]]
from = "/api/analyze"
to = "/.netlify/functions/analyze-video"
status = 200
force = true

[프론트엔드 / UX / 디자인 요구사항]

프론트엔드 전체는 “실제 상업용 AI SaaS 제품”처럼 보여야 합니다. 데모 느낌이 아니라, 투자자나 일반 사용자에게 곧바로 보여줄 수 있는 수준의 UI를 목표로 해 주세요.

기본 기술: React + TypeScript + Vite + Tailwind CSS
SPA 라우팅(예: react-router)을 사용해서 최소 두 화면을 구성합니다.

메인 화면(/) – 제품 소개 + 링크 입력 전용

메인 페이지는 최대한 단순하게, 링크 입력 한 가지 행동에 집중한 화면이어야 합니다.

배경:
고급스러운 다크 톤. 단색 검정이 아니라, 아주 짙은 네이비와 블랙이 섞인 부드러운 그라데이션 배경을 사용해 주세요. 화면 중앙 주변에는 약한 라디얼 하이라이트를 주고, 전반적으로 살짝 안개 낀 듯한 고급 SaaS 느낌이 나도록 Tailwind의 bg-gradient, from/via/to 클래스를 활용해 주세요.

상단:
얇은 네비게이션 바. 왼쪽에 단순 텍스트 로고(예: “AI Video Inspector”), 오른쪽에 가벼운 링크 두세 개(How it works, Docs, Sign in placeholder 정도)를 배치하되, 메인 액션을 방해하지 않도록 미니멀하게 스타일링합니다. 네비게이션 아래에는 얇은 알림 바를 두고, “Beta · 분석 결과는 확률적 추정이며 100% 정확하지 않을 수 있습니다.” 같은 한 줄 안내를 작게 표시해 주세요.

제품 소개 섹션:
네비게이션 아래에 간단한 제품 소개 영역을 둡니다. 강렬한 헤드라인 한 줄(예: “AI가 만든 영상인지, 한 번에 확인하세요.”)과 짧은 서브 텍스트 두 줄 정도로, “YouTube 링크를 넣으면 화면 기반 딥러닝 탐지기로 AI 생성 가능성을 분석해 주는 서비스”라는 가치 제안을 설명해 주세요. 오른쪽 또는 아래에는 아이콘 + 짧은 텍스트로 된 키 포인트 3개 정도를 배치해 주세요. 예를 들어 “딥러닝 기반 탐지”, “화면·프레임 분석”, “설명 가능한 리포트” 같은 걸 카드 형태로 보여주면 됩니다.

중앙 입력 카드:
화면 중앙에는 하나의 입력 카드만 떠 있는 구조로 만들어 주세요. 이 카드는 약간 유리 패널(glassmorphism) 느낌으로, 배경보다 조금 더 밝고, 살짝 블러와 그림자를 가진 스타일이면 좋습니다. 카드 안에는 “YouTube 링크를 입력하세요” 라벨, URL 입력 필드, “분석하기” 버튼만 넣어 주세요. 불필요한 부가 텍스트는 배제합니다. 버튼 클릭 시 로딩 상태로 바뀌고, 유효하지 않은 URL인 경우에는 카드 안 아래쪽에 짧은 에러 메시지를 표시합니다.

메인 페이지에서 “분석하기”를 누르면 SPA 라우팅으로 /analysis 경로로 이동하고, 그곳에서 분석 결과를 보여주는 구조로 해 주세요. 즉, / 는 입력 전용, /analysis 는 결과 전용입니다.

반응형:
데스크탑에서는 상단 소개 섹션 + 중앙 카드가 여유 있는 여백과 함께 정렬되고, 모바일에서는 소개 텍스트가 위, 입력 카드가 아래로 자연스럽게 쌓이도록 Tailwind의 responsive 클래스를 사용해 주세요.

분석 결과 화면(/analysis) – 상업용 리포트 느낌

분석 결과 페이지는 첨부한 스크린샷과 비슷한 구조를 가지되, 다음과 같은 개선 사항을 반영해 주세요.

전체 레이아웃:

최상단에 “분석 중인 영상 정보 바”를 둡니다.
왼쪽에는 작은 썸네일 아이콘과 함께 분석 대상 YouTube URL 또는 제목 일부를 보여주고, 그 아래에 작은 메타 정보를 추가해 주세요. 예: 채널명, 영상 길이, “분석 기준: 썸네일 + 프레임 1장” 같은 텍스트. 오른쪽에는 “다른 영상 분석하기” 버튼을 배치해, 클릭 시 / 경로로 이동하게 합니다.

그 아래에는 큰 메인 카드 하나를 중앙에 배치합니다. 이 카드 안은 좌우 2열로 구성합니다.
왼쪽: 영상 썸네일 프리뷰. 썸네일 아래에 “분석 대상 영상 썸네일 · YouTube Shorts · 0:17 · @채널명” 같은 짧은 캡션을 넣어 주세요. 호버 시 살짝 확대되면서 그림자가 짙어지고, 오른쪽 아래에 “영상 열기” 아이콘이 나타나는 정도의 마이크로 인터랙션을 추가해 주세요.
오른쪽: 큰 원형 게이지 또는 반원형 게이지 컴포넌트. 중앙에는 20 같은 숫자와 함께 percent가 보이고, 숫자 바로 아래에는 “AI 생성일 가능성 20%”처럼 이 값이 정확히 무엇을 의미하는지 한 줄로 명확히 적어 주세요. 그 아래에는 작은 텍스트로 “이 영상은 실제 촬영 영상일 가능성이 높습니다.”처럼 label을 자연어로 해석한 문장을 추가합니다.

게이지 색상 규칙:
점수가 낮을수록(0~30) 파랑/민트 계열, 중간(30~70)은 노랑/주황 계열, 높을수록(70~100)은 레드 계열이 되도록 Tailwind 색상을 조합해 주세요. 게이지 주변에는 색감이 살짝 퍼져나가는 글로우를 줘서 시각적인 집중도를 높입니다. 게이지 아래에는 작은 배지 형태의 상태 표시를 달아 주세요. 예: “Likely Authentic”, “Unclear”, “Likely AI” 등. 이 배지는 라운드 pill 형태의 버튼처럼 보이되 클릭 가능하지는 않은 상태 표시입니다.

메인 카드 아래 영역은 두 개의 카드로 2열 레이아웃을 만듭니다.
왼쪽 카드는 “분석 근거” 섹션, 오른쪽 카드는 “확인 팁” 섹션입니다.

분석 근거 카드:

상단에 아이콘(예: 돋보기/그래프)과 함께 “분석 근거” 제목, 그 아래에 “탐지된 주요 지표” 같은 짧은 부제목을 넣어 주세요.

콘텐츠는 각 항목이 3단 구조가 되도록 합니다.
(1) 작은 카테고리 태그: [색상 패턴], [텍스처], [노이즈], [딥러닝 결과] 등
(2) 굵은 제목: “색상 분포가 매우 균등함”
(3) 한 줄 또는 두 줄 설명: “화면 전체에 색이 고르게 퍼져 있어 실제 촬영 영상보다 인위적인 느낌이 있습니다. 생성형 모델에서 자주 관찰되는 특성입니다.”

항목 왼쪽에는 1, 2, 3… 번호가 들어간 작은 원형 배지를 두어 리스트 구조를 명확하게 보여 주세요.

전체 카드는 부드러운 그림자와 라운드 코너를 가지고, hover 시 살짝 떠올라 보이도록 shadow를 강화하는 정도의 인터랙션을 추가합니다.

확인 팁 카드:

상단에 전구 아이콘과 함께 “확인 팁” 제목, 아래에 “추천 다음 단계” 같은 부제목을 넣어 주세요.

컨텐츠는 버튼 같은 박스 스타일의 체크리스트 항목들입니다.
예:
“분석 결과 자연스러운 영상으로 보입니다.”
“그래도 중요한 정보라면 다른 출처와 교차 확인하세요.”
“제작자의 신뢰도도 함께 고려하세요.”

각 항목은 얇은 테두리와 라운드 처리된 박스로 렌더링되고, hover 시 배경색과 그림자가 약간 바뀌도록 해 주세요. 왼쪽에는 작은 아이콘(체크 표시나 화살표)을 넣어 시각적으로 클릭해 보고 싶어지는 느낌을 주되, 실질적으로는 단순 정보 카드입니다.

하단 안내 문구:

페이지 하단 또는 결과 카드 아래에 작게 다음과 비슷한 한 줄 안내를 넣어 주세요.
“이 분석 결과는 통계적 추정치이며, 100% 정확하지 않을 수 있습니다. 중요한 판단에는 다른 정보와 출처를 함께 참고해 주세요.”

이 문장은 과하게 눈에 띄지 않지만, 사용자가 찾으려 하면 쉽게 볼 수 있는 위치에 배치해 주세요.

상태 처리:

분석 중(/analysis 로 이동해서 백엔드 호출 직후)에는 메인 카드와 아래 2열 카드 대신 Skeleton UI를 보여 주세요. 카드 윤곽과 흐릿한 바 애니메이션 정도로, 상업 서비스에서 흔히 보이는 로딩 스켈레톤을 Tailwind로 구현합니다.

에러 발생 시에는 결과 대신 에러 카드 하나를 중앙에 보여주세요.
“분석 중 문제가 발생했습니다. 유효한 YouTube 링크인지 확인한 뒤 다시 시도해 주세요.” 같은 메시지를 표시하고, “처음으로 돌아가기” 버튼을 함께 제공해 / 경로로 이동할 수 있게 합니다.

컴포넌트 구조:

src/components에 Layout, Header, Footer, MainInputCard, AnalysisLayout, ScoreGauge, ReasonList, TipsList, MetaInfoBar, LoadingSkeleton, ErrorState 등의 컴포넌트를 나눠서 작성해 주세요.

src/hooks/useAnalyzeVideo.ts 커스텀 훅을 만들어, 입력 URL 상태, 로딩/에러 상태, AnalyzeVideoResponse 결과를 관리하게 해 주세요.

타입은 src/types/analysis.ts에서 import해서 사용합니다.

모든 UI 요소는 Tailwind 클래스를 난잡하게 나열하기보다는, 재사용 가능한 패턴과 공통 스타일을 염두에 두고 구성해 주세요. 전체적으로 “고급 다크 테마 + 카드 기반 리포트 + 명확한 숫자/라벨 + 여백이 충분한 레이아웃”이라는 느낌이 코드와 UI 모두에서 잘 드러나도록 구현해 주면 좋겠습니다.

마지막으로, 전체 프로젝트는 Replit에서 npm install 후 npm run dev로 바로 실행 가능해야 하고, Netlify에 바로 연결해 빌드/배포할 수 있는 수준의 완성된 예시 프로젝트여야 합니다.
